Redis基本知识

  1. 测试redis的并发处理能力 命令：redis-benchmark

  2. 查看redis服务是否正常运行 命令：ping, 如果正常返回pong

  3. 查看redis服务器的统计信息 命令： info [信息段]

  4. redis的数据库实例：redis中的数据库实例只能由redis来创建和维护，不可修改和自定义数据库实例。默认情况下，redis创建16个数据库实例。这些数据库实例以编号0~15识别。

    可以通过配置文件修改redis启动时的数据库实例个数。redis中的数据库实例本身占用存储空间很少

    默认情况，redis连接0号数据库实例；通过命令：select index 选择其他数据库实例

  5. 查看当前数据库key的数量： 命令：dbsize

  6. 查看当前数据库实例所有key的实例， **命令：keys ***

  7. 清空当前数据库实例： flushdb

  8. 清空所有数据库实例：flushall

  9. 查看redis中的配置信息。 **config get ***

Redis的数据结构

​ redis除了基本的key-value以外， 还包括 string, list, set， hash， zset；这些都是对value而言。

​ 其中，hash 单key: 对象（属性：值）

redis的操作命令

  1. 对于key的操作命令：**(斜体字表示具体数值)**

    • keys pattern, pattern表示通配符。
      • 如：keys * / ? / []
      • keys h[abc]llo, 表示以h开头，llo结尾，并从abc中选择一个字符
    • 判断key是否存在：exists key。 如果存在，返回1，否则返回0
    • 移动key到指定的数据库实例：move key index
    • 查看key的剩余生存时间： ttl key
      • 返回值为-2， 表示没有这个key
      • 返回值为-1， 表示没有设置生存时间，一直存在
    • 给key设置最大生存时间：expire key seconds
    • 查看指定key的数据类型：type key
    • 删除key: del key。返回实际删除的数量

  2. 操作string类型的命令

    • 设置string类型值到redis中：set key value。同名key后者会覆盖前者
    • 追加字符串：append key value。返回追加之后字符串的长度
    • 获取字符串数据的长度：strlen key
    • 将字符串数值进行加1运算：incr key
      • 如果key不存在，创建一个新key, 初始值为0， 然后再加1
      • 前提：key表示的value所指的字符串必须是一个数值字符串
    • 将字符串数值进行减1运算：decr key
    • 通过偏移量offset修改字符串：incrby/decrby key offset
    • 截取字符串：getrange key startIndex endIndex。采用闭区间
      • 下标可以是负，表示倒序取值。最右边为-1，对比Python
    • 用value覆盖starIndex开始的字符串： setrange key starIndex value
    • 设置字符串数据的同时设置最大生命周期：setex key seconds value
    • 当key不存在的时候才会设置，避免覆盖情况出现：setnx key value
    • 批量设置string类型的数据到redis中：mset key1 value1 key2 value2 ….
    • 类似的，批量获取：mget key1 key2 ….

  3. 操作List类型的命令

    • 将一个或多个值一次放入list的表头：lpush key value [value vlaue]
    • 查看list中的数据, 可以指定下标区间：lrange key startIndex endIndex
    • 将一个或多个值一次放入list的表头：rpush key value [value vlaue]
    • 从表头list中删除数据, 并返回表头元素：lpop key
    • 从list中删除数量count的和value相等的数据数据：lrem key count value；
      • 默认从右侧删除，如果count<0，则从左侧进行删除；count == 0，表示删除所有与value相等的值
    • 获取list中指定下标的元素：lindex key index
    • 获取list的长度：llen key
    • 截取List中指定下标区间元素组成新的列表：ltrim key startIndex endIndex
    • 将指定位置元素设置为新的值：lset key index value
    • 将value插入到list中pivot的前或后： linsert key before/after pivot value

  4. 操作set类型的命令：

    set中的数据无序不可重复

    • 添加一个或多个元素：sadd key value [value value]
    • 获取set中所有元素：smembers key
    • 判断元素在集合中是否存在：sismember key value
      • eg：sismember set01 f; 存在返回1， 不存在返回0
    • 获取集合长度：scard key
    • 移除集合中的元素：srem key value [value …]
    • 随机获取集合中的一个元素：srandmember key [count]
      • count用于指定随机的元素数量，当count > 0多个元素之间不重复，否则，可以重复
    • 随机移除一个元素：spop key[count]
    • 移动集合到新集合中：smove source dest member
    • 获取差集：sdiff key1 key [key …]
    • 交集：sinter key1 key [key …]
    • 并集：sunion key1 key [key …]

  5. 操作hash类型的命令：

    采用key和多value键值对存储，其中value又表示一系列键值对（field-value），用来存储对象

    • 设置一个或多个数据到hash中：hset key field1 value1 [field2 value2] …
    • 获取hash中指定key的指定field和value：hget key field
    • 批量获取field值：hmget key field1 [field1 field2]
    • 获取所有的field: hgetall key
    • 删除hash中一个或多个field：hdel key field1 [field2…]
    • 获取field的个数：hlen key
    • 判断hash中是否存在某一个field：hexists key field
    • 获取所有的field列表：hkeys key
    • 获取所有的values列表：hvals key
    • 设置value不覆盖原来的值：hsetnx key field value

  6. 操作zset类型的命令：

    zset: 存入value的时候自带score，根据score进行从小到大排序，表示可以排序的集合；即有序不可重复；

    • 添加一个或多个member及其score： zadd key score member [score member1]
      • 如果元素存在，覆盖分数score
    • 获取zset中的区间元素：zrange key starIndex endIndex[withscores]
      • 带有withscores表示同时显示score和value
    • 根据分数获取闭区间长度的元素：zrangebyscores key min max [withscores]
    • 删除一个或多个元素：zrem key member [member]
    • 获取指定集合元素个数：zcard key
    • 获取指定分数区间的元素个数：zcount key min max
    • 获取zset中元素的排名(排名从0开始)：zrank key member
    • 获取zset中元素的排名(排名降序获取)： zrevrank key member
    • 获取指定元素的分数：zscore key member

Redis的配置文件

​ redis配置文件在安装路径下存在redis.conf;配置redis服务运行时的一些参数；修改配置文件后，使用redis服务必须指定参数才可以使用配置后的参数

  • 网络配置

    • port:指定redis服务使用的端口号， 默认6379
    • bind：客户端连接redis服务所能使用的ip地址， 默认可以使用redis服务所在主机上的任意ip； 一般会进行配置为一个真实ip

  • 常规配置

    • logfile:指定日志文件。默认情况输出到控制台，可以指定配置文件的位置
    • databases：数据库实例，默认16，可以自己修改

Redis持久化

​ Redis为内存数据库，数据一般存储在内存中。但为了防止数据丢失，提供持久化功能：RDB和AOF，在适当的时机将内存数据持久化到磁盘上，每次redis服务的启动会将磁盘数据加载到内存

​ 1.RDB策略：在指定时间间隔内，redis服务执行指定次数的写操作，会自动触发一次持久化操作

​ | RDB策略是redis默认启动的策略，服务启动的时候默认开启

​ | RDB配置：dbfilename:表示持久化生成的文件名，默认 dump.rdb； dir：RDB持久化策略生成文件的保存位置

  1. AOF策略：弥补RDB可能造成的断档数据丢失问题。采用操作日志记录每一次写操作；但是由于需要写磁盘，效率低，默认不开启

实际中，一般只开启RDB，因为数据最开始一般放在关系型数据库中，对于访问频率高的数据会放入到redis，但是在关系型数据库中仍有备份。所以redis中追求效率更重要

Redis事务

​ 允许将一组命令放到一起序列化，保证这些指令按顺序执行，仅保证部分原子性

  1. mutli:用来标记一个事务的开始。事务开启后的命令首先会放入一个队列中，当事务提交的时候，在执行队列中的指令
  2. exec：用来执行队列中的所有指令
  3. watch：用来监控某一vlaue值，如果在事务开启阶段被其他人更改，那么当前整个事务都不会执行
  4. unwatch：解除所有被监视的对象

Redis消息的发布与订阅

redis客户端订阅频道，发布者在该频道上的信息会被订阅者接受；用于处理当多个客户端需要交互的情况

  1. subscribe:订阅一个或多个频道的信息
  2. publish:将消息发送到指定频道

Redis的主从复制

​ 用于redis集群环境，多台redis服务器负责不同的不同，一部分读，另一部分写；负责写机制的作为主库，读机制的负责从库， 主少从多。当主库写入新数据后会自动同步到从库；实现读写分离，减少高并发的访问压力

​ 通过一台redis修改config模拟多台redis服务，默认情况下每个redis服务都是master，读写数据不会影响；可以通过info replication查看每个redis的主从角色；在需要的服务上使用salveof可以指定当前redis服务为从库

从库不可以只读：

全量复制：一旦主从关系确定，会自动把主机上的已有数据同步复制到从库

增量复制：主机上写数据会自动同步到从库

意外情况：主机宕机，从机原地待命；主机恢复，一切正常

​ 从机宕机，主机减少一个slave，其他从机一切正常；从机恢复，必须重新设置从属关系

从机上位：从机翻身做主

​ 先通过slaveof no one 改变从机的主从角色；改变其他从机的主从关系重新设置主从关系

从机之间也可以有从属关系，减少主机的压力，但增大了服务的延迟
